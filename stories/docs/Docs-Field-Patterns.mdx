# Field Patterns

This guide shows how to wire **labels**, **descriptions/help**, and **errors** so screen readers announce them correctly.

## The anatomy

- **Label** — connect via `htmlFor` to the input `id`.
- **Description/Help** — provide extra context; connect with `aria-describedby`.
- **Error** — show when invalid; include it in `aria-describedby` and set `aria-invalid="true"`.

## Pattern: Input with help + error

```tsx
function ExampleInputField() {
  const [value, setValue] = React.useState("");
  const [touched, setTouched] = React.useState(false);
  const invalid = touched && value.trim().length < 3;

  const inputId = "name-input";
  const helpId = "name-help";
  const errorId = "name-error";

  const describedBy = [
    invalid ? errorId : null,
    helpId
  ].filter(Boolean).join(" ");

  return (
    <div style={{ display: "grid", gap: 6, maxWidth: 360 }}>
      <label htmlFor={inputId} style={{ fontWeight: 600 }}>Display name</label>
      <input
        id={inputId}
        className="aui-input"
        value={value}
        onChange={(e) => setValue(e.target.value)}
        onBlur={() => setTouched(true)}
        aria-invalid={invalid || undefined}
        aria-describedby={describedBy}
        placeholder="At least 3 characters"
      />
      <div id={helpId} className="aui-help" style={{ color: "var(--aui-color-text-muted)" }}>
        This will be visible on your profile.
      </div>
      {invalid && (
        <div id={errorId} className="aui-error" style={{ color: "var(--aui-color-danger)" }}>
          Name must be at least 3 characters.
        </div>
      )}
    </div>
  );
}
